# Matriz Heurística

Dentro del análisis de la pantalla principal del aplicativo se encontraron los siguientes fallos dentro del sistema correspondientes a la pantalla inicial o dashboard de ingreso.

| Heurística | Pantalla | Problema detectado | Severidad |
|---|---|---|---|
| H2 - Relación entre el sistema y el mundo real | Dashboard principal | La gráfica correspondiente a evolución de test de los últimos 30 días cuenta con etiquetas en inglés mientras que todo el sistema se encuentra en español. | Media |
| H8 - Diseño estético y minimalista | Dashboard principal | Existe una sobrecarga de datos que puede sobre estimular con información al usuario ya que presenta datos sobre los principales test, pero específicos sobre solo el que se está presentando en las tarjetas. | Baja |
| H3 - Control y libertad del usuario | Dashboard principal | Al exportar el PDF correspondiente al resumen ejecutivo se muestra en una pantalla externa a la correspondiente dentro del sistema, obligando al usuario a salir del flujo del aplicativo. | Alta |
| H4 - Consistencia y estándares | Dashboard principal (Métricas) | El indicador de “Tiempo Promedio” muestra un valor negativo (-10%) en color rojo. En rendimiento, reducir el tiempo suele ser positivo, por lo que el uso del color rojo contradice la semántica del éxito en el dominio del software. | Media |
| H7 - Flexibilidad y eficiencia de uso | Dashboard principal (Planes de test) | Las pestañas de filtrado (Borradores, En progreso, etc.) no muestran contadores numéricos. El usuario debe clicar en cada una para saber si hay contenido, aumentando el esfuerzo para obtener un resumen rápido. | Baja |
| H1 - Visibilidad del estado del sistema | Dashboard principal (Gráficas) | Las gráficas de líneas en las tarjetas de métricas carecen de etiquetas de datos o puntos de información (tooltips). El usuario ve la tendencia, pero no puede conocer el valor exacto en un punto específico del tiempo. | Media |
| H8 - Diseño estético y minimalista | Dashboard principal (General) | El contraste de las etiquetas de texto pequeñas (como “3 TOTAL” o los ejes de la gráfica de barras) es demasiado bajo respecto al fondo, dificultando la legibilidad y violando principios de accesibilidad visual. | Baja |
| H1 - Visibilidad del estado | Gestión de Grupos | Duplicidad de selectores: Existe un selector de “Grupo de trabajo” en la barra superior y otro en el cuerpo lateral. No queda claro cuál manda sobre el contenido o si cambiar uno actualiza el otro automáticamente. | Moderado |
| H4 - Consistencia y estándares | Gestión de Grupos | Jerarquía de botones: El botón “Crear nuevo grupo” (azul oscuro) y el botón “Nuevo Test” (blanco) en el header compiten visualmente. No se distingue cuál es la acción primaria global del aplicativo. | Leve |
| H3 - Control y libertad | Configuración | Falta de confirmación de eliminación: El icono de la papelera en la lista de miembros está muy cerca del borde y parece activarse con un solo clic. Sin un diálogo de confirmación, un error puede expulsar a un usuario accidentalmente. | Crítico |
| H7 - Flexibilidad y eficiencia | Configuración | Botón “Añadir” inactivo/confuso: El botón de añadir miembro parece deshabilitado visualmente (azul claro) a pesar de que hay texto de ejemplo en el input. Debería resaltar claramente cuando el formato de correo es válido. | Moderado |
| H8 - Diseño estético | Gestión de Grupos | Uso excesivo de tarjetas blancas sobre fondo gris muy claro: La falta de sombras más definidas o bordes hace que las tarjetas de los grupos se pierdan un poco visualmente en pantallas con poco contraste. | Leve |
| H5 - Prevención de errores | Configuración | Botón de retroceso ambiguo: La flecha hacia atrás junto a “Los wachiturros” es pequeña y está aislada. El usuario podría dudar si vuelve a la lista de grupos o si cancela los cambios realizados en esa pantalla. | Leve |
| H4 - Consistencia y estándares | General | Mezcla de idiomas en nombres: El sistema está en español, pero el nombre de usuario arriba a la derecha no tiene opciones de “Perfil” o “Configuración” en un menú desplegable claro, mezclando iconos funcionales con texto estático. | Leve |
| H6 - Reconocimiento vs Recuerdo | Gestión de Grupos | Iconografía repetitiva: El icono de la “empresa/edificio” se usa tanto para el grupo como para el botón de configurar. No ayuda a diferenciar visualmente qué es un objeto de datos y qué es una acción. | Leve |
| H1 - Visibilidad del estado | Configuración | Feedback de rol no interactivo: Las etiquetas “ADMIN” y “MIEMBRO” parecen botones, pero son solo etiquetas de texto. El usuario podría intentar hacer clic para cambiar el rol y frustrarse al ver que no sucede nada. | Moderado |
| H2 - Relación con el mundo real | Gestión de Grupos | Nomenclatura en la barra lateral: Los nombres de los grupos en el menú lateral aparecen con un icono de escudo, mientras que en el centro tienen un icono de edificio. Esta inconsistencia visual genera duda sobre si se trata del mismo elemento. | Leve |
